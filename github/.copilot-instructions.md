# Copilot Coding Rules for This Project

## ‚ùó ABSOLUTE REQUIREMENTS
Copilot MUST follow these rules at all times when generating code for this repository:

### 1. **NO TYPESCRIPT ‚Äî NEVER**
- Do NOT generate `.ts` files.
- Do NOT use TypeScript syntax (no types, no interfaces, no generics).
- Do NOT generate JSDoc with type annotations.
- Do NOT import or reference TypeScript-only libraries.
- ALL code must be plain JavaScript.

### 2. **Svelte 5 ONLY**
All UI code must use the new Svelte 5 syntax (Runes):
- `$state`
- `$derived`
- `$effect`
- No old Svelte 3/4 syntax like `export let`, `$:` reactive labels, etc.

### 3. **File Types**
Allowed:
- `.js`
- `.svelte`
- `.json`
- `.md`

Not allowed:
- `.ts`
- `.d.ts`
- `*.ts` inside `<script>` blocks.

### 4. **Backend / Server**
If backend code is generated:
- Must be plain JavaScript.
- No TypeScript req/res types.
- Use native Node.js or SvelteKit JavaScript API only.

## üß≠ Output Format Rules
- Provide COMPLETE snippets.
- Avoid placeholder types or ‚ÄúTODO: add types‚Äù.
- Use JavaScript objects, arrays, and functions only.
